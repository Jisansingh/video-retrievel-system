# Video Retrieval System

Multimodal video retrieval system using **OpenAI CLIP ViT-B/32** embeddings and **FAISS** similarity search. Supports **text-to-video**, **image-to-video**, and **video-to-video** retrieval via a FastAPI backend with a React + Vite frontend.

---

```
                               └─────────────────────────┘
```

**Retrieval workflow**:

```
Text Query ──→ CLIP Text Encoder ──→ FAISS Search ──→ Ranked Videos
Image Query ──→ CLIP Image Encoder ──→ FAISS Search ──→ Ranked Videos
Video Query ──→ Frame Sampling (every 30th) ──→ CLIP Encoder ──→ Mean Pool ──→ FAISS Search ──→ Ranked Videos
```

## Features

### Frontend

| Feature | Description |
|---|---|---|
| About page | Project overview, tech stack, capabilities |
| Explore Library | Browse all 500 videos grouped by 5 categories, paginated grid with Load More |
| Search page | Text/image/video/frame query tabs with top-K selector |
| History page | Persisted search history in localStorage with re-run capability |
| Video preview modal | HTML5 video player with backdrop blur, filmstrip, playback speed |
| Frame search | Text-to-frame retrieval with timestamp + category metadata |
| Search history persistence | Stored in localStorage under `searchHistory` key |

### Search

| Mode | Endpoint | Description |
|---|---|---|---|
| Text-to-video search | `GET /search` | Natural-language query, CLIP text encoder |
| Text-to-frame search | `GET /search/frames` | Natural-language query, frame-level CLIP search |
| Image search | `POST /search` | Upload image, CLIP image encoder |
| Video search | `POST /search/video` | Upload video, frame sampling + mean pooling |
| Frame download | `GET /download/frame` | Download a matched frame as JPEG |
| Similarity scoring | — | Cosine similarity (L2 norm + inner product), scores 0–1 |
| Top-K retrieval | — | Configurable 5/10/20/50 results |

### Backend

| Component | Technology |
|---|---|
| REST API | FastAPI + Uvicorn |
| Embedding model | OpenAI CLIP ViT-B/32 (512-d) |
| Similarity index | FAISS `IndexFlatIP` (brute-force inner product) |
| Video serving | Static file mount on `/videos/*` |
| Thumbnail serving | Static file mount on `/frames/*` |

## Dataset Structure

```
data/
├── videos/                    # Source video files
│   ├── animals/               # ~100 videos
│   ├── humans/                # ~100 videos
│   ├── indoors/               # ~100 videos
│   ├── nature/                # ~100 videos
│   └── vehicles/              # ~100 videos
├── frames/                    # Extracted frame thumbnails
│   ├── animals/
│   │   └── animals_14/
│   │       ├── frame_00000.jpg
│   │       └── ...
│   └── ...
├── embeddings/                # Auto-generated by ML pipeline
│   ├── video_embeddings.npy   # (N, 512) float32 array
│   ├── video_metadata.json    # N video path strings
│   ├── frame_embeddings.npy   # Per-frame CLIP embeddings
│   └── frame_metadata.json    # Frame-level metadata (path, video, timestamp, category)
└── indexes/                   # Auto-generated by ML pipeline
    ├── video_index.faiss      # Video-level FAISS IndexFlatIP
    ├── video_metadata.json    # Metadata copy for backend
    ├── frame_faiss.index      # Frame-level FAISS IndexFlatIP
    └── frame_metadata.json    # Frame metadata copy for backend
```

The demo dataset contains **500 categorized videos** (100 per category) across 5 categories: animals, humans, indoors, nature, and vehicles.

## API Documentation

### `GET /`

Root endpoint. Confirms the server is running.

**Example response:**
```json
{
  "message": "Video Retrieval System Running"
}
```

### `GET /health`

Liveness probe.

**Example response:**
```json
{
  "status": "ok"
}
```

### `GET /library`

List all videos grouped by category. Built dynamically from the filesystem.

**Example response:**
```json
{
  "animals": [
    {
      "title": "animals_14.mp4",
      "category": "animals",
      "video_url": "/videos/animals/animals_14.mp4",
      "thumbnail_url": "/frames/animals/animals_14/frame_00000.jpg"
    }
  ],
  "humans": [],
  "indoors": [],
  "nature": [],
  "vehicles": []
}
```

### `GET /search` — Text-to-Video

Search for videos matching a natural-language description.

**Query Parameters:**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `query` | `string` | (required) | Natural-language search query (min 1 char) |
| `top_k` | `int` | `5` | Number of results (1–50) |

**Example request:**
```bash
curl "http://localhost:8000/search?query=a+dog+running+on+grass&top_k=3"
```

**Example response:**
```json
{
  "query": "a dog running on grass",
  "results": [
    {
      "rank": 1,
      "video": "animals/animals_14",
      "category": "animals",
      "score": 0.2761,
      "title": "animals_14.mp4",
      "video_url": "/videos/animals/animals_14.mp4",
      "thumbnail_url": "/frames/animals/animals_14/frame_00000.jpg"
    }
  ]
}
```

### `POST /search` — Image-to-Video

Upload an image and retrieve visually similar videos.

**Request** (multipart/form-data):

| Field | Type | Description |
|---|---|---|
| `image` | file | JPEG, PNG, WebP, BMP, or TIFF (max 10 MB) |
| `top_k` | int (query) | Number of results (1–50, default 5) |

**Example request:**
```bash
curl -X POST "http://localhost:8000/search?top_k=3" \
  -F "image=@query_image.jpg"
```

**Example response:**
```json
{
  "query": "query_image.jpg",
  "results": [
    {
      "video": "animals/animals_14",
      "score": 0.2761,
      "title": "animals_14.mp4",
      "category": "animals",
      "video_url": "/videos/animals/animals_14.mp4",
      "thumbnail_url": "/frames/animals/animals_14/frame_00000.jpg"
    }
  ]
}
```

### `POST /search/video` — Video-to-Video

Upload a video and retrieve similar videos. The uploaded video is sampled (every 30th frame), each frame is encoded with CLIP, and frame embeddings are mean-pooled into a single query vector.

**Request** (multipart/form-data):

| Field | Type | Description |
|---|---|---|
| `video` | file | MP4, AVI, MOV, MKV, or WebM |
| `top_k` | int (query) | Number of results (1–50, default 5) |

**Example request:**
```bash
curl -X POST "http://localhost:8000/search/video?top_k=3" \
  -F "video=@query_video.mp4"
```

**Example response:**
```json
{
  "query": "query_video.mp4",
  "results": [
    {
      "video": "animals/animals_14",
      "score": 0.8910,
      "title": "animals_14.mp4",
      "category": "animals",
      "video_url": "/videos/animals/animals_14.mp4",
      "thumbnail_url": "/frames/animals/animals_14/frame_00000.jpg"
    }
  ]
}
```

### `GET /videos/*` — Static Video Serving

Serves video files directly from the `data/videos/` directory.

**Example:**
```
http://localhost:8000/videos/animals/animals_14.mp4
```

### `GET /frames/*` — Static Thumbnail Serving

Serves frame thumbnails directly from the `data/frames/` directory.

**Example:**
```
http://localhost:8000/frames/animals/animals_14/frame_00000.jpg
```

### `GET /search/frames` — Text-to-Frame Search

Search individual frames matching a natural-language description. Returns frame-level results with timestamps.

**Query Parameters:**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `query` | `string` | (required) | Natural-language search query (min 1 char) |
| `top_k` | `int` | `12` | Number of frame results (1–50) |

**Example request:**
```bash
curl "http://localhost:8000/search/frames?query=a+dog+running&top_k=3"
```

**Example response:**
```json
{
  "query": "a dog running",
  "results": [
    {
      "frame_url": "/frames/animals/animals_14/frame_00005.jpg",
      "video": "animals/animals_14",
      "video_url": "/videos/animals/animals_14.mp4",
      "timestamp": 5.0,
      "category": "animals",
      "score": 0.3124
    }
  ]
}
```

### `GET /download/frame` — Frame Download

Download a matched frame as a JPEG file. Validates the path to prevent directory traversal.

**Query Parameters:**

| Parameter | Type | Description |
|---|---|---|
| `path` | `string` | Relative frame path (e.g. `animals/animals_14/frame_00005.jpg`) |

**Example request:**
```bash
curl -o frame.jpg "http://localhost:8000/download/frame?path=animals/animals_14/frame_00005.jpg"
```

## Environment Configuration

Create `frontend/.env`:

```
VITE_API_URL=http://localhost:8000
```

This tells the React frontend where to reach the FastAPI backend. The value is imported at runtime via `import.meta.env.VITE_API_URL`.

## Troubleshooting

### Backend fails to start

```
FileNotFoundError: FAISS index not found at ...
```

Run the ML pipeline first to generate the required index and metadata files:

```bash
python ml_pipeline/extract_frames.py
python ml_pipeline/generate_embeddings.py
python ml_pipeline/build_faiss.py
```

### `KMP_DUPLICATE_LIB_OK` error on macOS

If you see `OMP: Error #15: Initializing libomp.dylib`, the server handles this automatically by setting `KMP_DUPLICATE_LIB_OK=TRUE` at startup. If you encounter it in other contexts, run:

```bash
export KMP_DUPLICATE_LIB_OK=TRUE
```

### Frontend shows blank page or CORS errors

1. Confirm the backend is running on port 8000.
2. Check `frontend/.env` contains `VITE_API_URL=http://localhost:8000`.
3. Clear the browser cache and reload.

### Frontend port already in use

Vite defaults to port 5173. If that port is occupied, Vite will suggest an alternative. Update the backend CORS config in `backend/main.py` if needed.

### No videos appear in the library

Ensure videos are placed in `data/videos/` organised by category subfolder (e.g. `data/videos/animals/`). Only `.mp4`, `.avi`, `.mov`, `.mkv`, and `.webm` files are recognised.
